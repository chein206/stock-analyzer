| 포트폴리오 탭 | 관심종목 평단가·수량 입력, 수익률 계산 | 쿠키 저장 |
| 스크리너 탭 | 127개 종목 병렬 스캔, 신호 점수 순 정렬 | ThreadPoolExecutor 16 workers |

## ⭕ 미구현 — 향후 개발 항목

### A. KIS API 고도화
```
우선순위: 높음
```
- [ ] **KIS 잔고 자동 연동** → 포트폴리오 탭 평단가·수량 자동입력 (지금은 수동)
- [ ] **호가창** → 실시간 매수/매도 호가 10단계 표시
- [ ] **체결내역 조회** → 내 매매 기록 확인
- [ ] **주문 기능** → 앱에서 직접 매수/매도 (시장가/지정가)
  - 주의: KIS 주문 권한 별도 신청 필요
  - 모의투자 → 검증 후 실전 전환 권장

### B. 자동매매 시나리오
```
우선순위: 높음 (사용자 요청)
난이도: 높음
```
- [ ] **시나리오 설정 UI**
  ```
  예) 42,000원 10주 매수
      48,400원 도달 → 50% 익절
      나머지 67,400원 목표 보유
      39,000원 이하 → 손절
  ```
- [ ] **조건 감시** (앱 열려있는 동안 30초 주기 체크)
- [ ] **조건 충족 알림** → 화면 알림 + 원클릭 주문 버튼
- [ ] 한계: Streamlit Cloud는 앱 닫히면 절전 → 완전 자동은 별도 서버 필요

### C. 콘텐츠 강화
```
우선순위: 중간
```
- [ ] **뉴스 요약** → 종목 관련 최신 뉴스 3~5개 + AI 요약
- [ ] **재무제표 상세** → PER/PBR/ROE 업종 평균 비교
- [ ] **배당 정보** → 배당수익률, 배당락일, 배당 이력

### D. UX 개선
```
우선순위: 중간
```
- [ ] **모바일 최적화** → 터치 친화적 UI, 작은 화면 레이아웃
- [ ] **알림 기능** → 관심종목 목표가/손절가 도달 시 알림
- [ ] **차트 인터랙티브** → 매수·손절·목표가 라인 드래그 조정
- [ ] **스크리너 저장** → 자주 쓰는 조건 저장

## 🏗 기술 스택

```
Frontend  : Streamlit (Python)
AI 엔진   : Claude Sonnet 4.5 (Anthropic)
실시간 데이터 : KIS Developers API (한국투자증권)
지연 데이터  : FinanceDataReader, yfinance, pykrx
배포      : Streamlit Cloud
저장소    : 브라우저 쿠키 (streamlit-cookies-controller)
캐시      : st.cache_data (30초~24시간 TTL)
병렬처리  : ThreadPoolExecutor (스크리너)
```

---

## 📌 개발 원칙

1. **KIS 실패 시 항상 fallback** — 앱이 멈추면 안 됨
2. **캐시 적극 활용** — 실시간(30s) / 분석(900s) / 스크리너(3600s)
3. **쿠키 싱글톤** — `CookieController()` 스크립트 실행당 1회만 생성
4. **`cur_analysis` 패턴** — AI 채팅 rerun 시 분석 화면 유지
5. **모의투자 먼저 검증** — 주문 기능 추가 시 반드시 모의 테스트 후 실전
